# v1.3.0 懸念事項（スコープ外メモ）

サムネイル修正（グリッド描画のアスペクト比 / EXIF Orientation対応）の調査中に見つかった、
今回のスコープ外とした懸念点の記録。

## 1. CMYK JPEG 非対応

`decode_image_bytes`（`src/fs/archive/decode.rs:22`）はCMYK JPEGを特別扱いせず`image` crateの
デフォルトデコーダ任せ。crate側の対応状況次第で色が破綻する可能性がある。

## 2. プログレッシブJPEG / インターレースPNG 未検証

`progressive` / `interlace` のハンドリングがコード上見当たらない（grep 0件）。`image` crateの
デコーダに完全依存しており、途中状態のデータでサムネ化される懸念は未検証のまま。

## 3. サムネイルキャッシュの検証がmtimeのみ

`probe_cached_thumb`（`src/cache.rs:1037`）はDBの値を即返し、その後`file_mtime`との比較のみで
再生成要否を判断（`src/cache.rs:887-890`）。内容ハッシュやETag等でのコンテンツ照合は無いため、
mtimeが変化しないファイル差し替え（同一パス・同一mtimeでの中身入れ替え等）ではキャッシュが
古いままになりうる。
