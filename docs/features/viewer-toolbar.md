# ビューアーツールバー再編（見開き移設・アイコン化・項目順序基盤）

todo (E)「見開きボタンをビューアーウィンドウに移動」の実装メモ。
単純な移設ではなくツールバーの再編として扱い、将来の項目追加に耐える基盤を同時に敷く。

## 背景と方針

ビューアー top bar（[view_reader.rs](../../src/view_reader.rs) `draw_top_bar`）は既に
ソート群・回転群・ウィンドウスロットF5〜F8で満杯。エクスプローラー側メニューバー
（[panels.rs](../../src/view_explorer/panels.rs)）の見開き群は6要素
（ページモード3択・1P戻す/進む・整列状態ラベル）あり、丸ごと移すと確実に溢れる。

採用方針は「移動＋間引き」の複合:

1. **移すのはページモード3択＋1P戻す/進む**（いずれもアイコン化して常設、
   キー4/5は維持）。テキスト文言の状態ラベルは持ち込まない
2. **状態表示の圧縮**。「整列中/+1Pずれ中」の文言はやめ、ページモードアイコン隣の
   コンパクト表示に置き換える（下記）
3. **アイコン化による幅圧縮**。テキストラベル付きチェックボックス2つ
   （回転を引き継ぐ・EXIF回転）が最大の幅消費源。日本語ラベルが最長級のため、
   アイコン化はi18n幅問題も同時に解決する

### 決定事項（2026-07 確定）

- ページモード3択はアイコン化（候補: ▯◧◨等、要グリフ監査）＋ホバーツールチップ
- 1P戻す/進むもアイコンボタンとしてビューアーbarに常設（右クリックメニュー移設案は不採用）
- ずれ状態表示: オフセット値（-1/0/+1、[spread_offset.rs](../../src/spread_offset.rs)）を
  「OFFSET」等の文言なしで表示。0は「0」のみ（矢印なし）、±1は綴じ方向による
  ページ進行方向を加味した矢印付き「←1」/「1→」。
  例: 見開き右（右綴じ・左へ進行）で+1なら「←1」、見開き左は逆向き。単ページ時は非表示
- fs_sort_bar（フルスクリーンホバーバー）にもページモード群を載せる（Vec共有で追加コストほぼゼロ）

## アイコンリソースの保有方法（シングルバイナリ堅持）

優先順:

1. **Unicodeグリフ**（第一候補）: 既存の`⟲⟳`と同方式。egui本体がUbuntu-Light・
   NotoEmoji・emoji-icon-fontをクレート内埋め込みしており追加リソース不要。
   [glyph_audit.rs](../../src/view_explorer/glyph_audit.rs)の監査テスト基盤を
   ツールバーアイコン候補にも拡張して両OSの豆腐チェックを機械化する
   （マーカーと違い単色マスクで良いため塗り率判定は不要、収録有無のみ）
2. **epaint手描き**: フォントに無い字形のみ。リソースゼロ・解像度非依存・テーマ色追従
3. include_bytes!によるPNG埋め込みは今回不使用（DPIスケール対応の手間に見合わない）

## 項目順序基盤（順序付きVec＋設定保存）

将来の「ライブ項目並べ替え」「溢れ時のオーバーフロー畳み込み」の両方が同じデータ構造に
乗るため、先に基盤だけ敷く。**今回は編集UIなし・実質固定順**。

- `ViewerBarItem`（項目ID enum）と順序`Vec<ViewerBarItem>`を定義し、top bar /
  fs_sort_barの描画をVec順ループに再構成する
- 状態ファイルにカンマ区切り文字列で1キー保存（`viewer_bar_order`等）。
  gui_config.rsの手書きkey=valueパーサに追随
- **前方互換の解決規則を最初から実装する**（ここを省くと将来項目追加時に二度手間）:
  - 未知ID → 無視（旧バージョンへ設定を持ち帰っても壊れない）
  - 設定に無いID → 既定位置に補完（新バージョンで項目が増えても自然に出現）
- 右寄せのウィンドウスロットF5〜F8は順序Vecの対象外（固定右寄せのまま）

## 将来拡張（今回やらない、この基盤の上に載る）

- **ライブ並べ替え**: egui 0.35標準の`dnd_drag_source`/`dnd_drop_zone`で外部依存なしに
  実装可能。誤操作防止に「並べ替えモード」トグルを挟む。設定ダイアログ内での
  ↑↓並べ替えが低コスト先行案
- **優先度付きオーバーフロー**: `ui.available_width()`で残幅を測り、入らない項目を
  `…`のmenu_buttonへ自動送り。順序Vecの後方から畳む＝順序がそのまま優先度になる
- 運用ルール: 新項目はバー直載せをデフォルトにしない。低頻度機能は画像右クリック
  メニュー（`spread_save_context_menu`系）か設定ダイアログへ

## エクスプローラー側の後始末

見開き群6要素の除去で`MENU_BAR_ORDER`は11→6項目に縮小。`menu_bar_items()`・
`activate_menu_button`・キーボードナビの見開き分岐も同時に削除する。
エクスプローラー側メニューバーの順序は const のまま（順序Vec基盤はビューアー専用）。
エクスプローラー側も並べ替え対象にするのは別工事（todo (I) キーボードオペレーション
対応との絡みで検討）。

## 実装時の残確認事項

- ずれ表示の矢印向きマッピング（綴じ方向×オフセット符号の4通り）は実装時に
  実表示で最終確認する
- アイコングリフの最終選定はglyph_audit拡張テストの両OS結果を見て確定する
